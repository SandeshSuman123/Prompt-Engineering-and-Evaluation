Why this is called an "evaluation pipeline"

An evaluation pipeline automates the process of testing an LLM prompt:

Generate or load test cases (the evaluation dataset).
Run the prompt on each test case.
Collect the model's output.
Grade the output (currently a placeholder score = 10; in a real system this could be another LLM, rule-based checks, or exact-match tests).
Save the results for analysis.

This lets you compare different prompts objectively. For example, if you create Prompt A and Prompt B, you can run both on the same dataset and compare their scores to determine which prompt performs better.
